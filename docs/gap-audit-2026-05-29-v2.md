## CRITICAL REGRESSIONS FOUND

### CR1 — QuoteWorkflowBuilder submit is NO LONGER async
The `onSubmit` on `<QuoteWorkflowBuilder>` is back to sync (`(total) => {…}`) instead of async. The API wiring was done in `ArtisanDashboardCoreSection`'s `onSubmit` handler but the `QuoteWorkflowBuilder` component's confirm button now passes the full params. **Confirm the `onSubmit?.(subtotal, items, depositPercent)` call and the async handler are connected.**

### CR2 — Client find/saved in source preview uses fixture
When an artisan/client user navigates to `/client/find` or `/client/saved` via the SOURCE PREVIEW sidebar nav, the internal `{view === "find"}` and `{view === "saved"}` views show `recommendedArtisans` fixture (public artisan cards). The REAL `ClientFindView` and `ClientSavedView` from `client-views.tsx` (with `useArtisanSearch` + `useSavedArtisansPage`) are only used when routing via the modular layout pages — which are now bypassed since all routing goes through SourceAdminPreview.
